# Single requirements file — covers CLI, GUI (PyQt6), online-fix (Selenium), and Tor fallback.
# Versions taken from requirements-consumer.txt (more recent); gui + online-fix merged in.
#
# ---------------------------------------------------------------
# INSTALLATION (two commands required due to steam urllib3 conflict)
#
#   Step 1:  pip install -r requirements.txt
#   Step 2:  pip install steam==1.4.4 --no-deps
#
# Why? steam==1.4.4 metadata declares urllib3<2, but selenium 4.x
# needs urllib3>=2.x. Steam only uses `requests` at runtime and works
# fine with urllib3 2.x — the constraint is just stale. Installing
# steam with --no-deps bypasses the check; all its deps are already
# installed by Step 1.
# ---------------------------------------------------------------
#
# ---------------------------------------------------------------
# OPTIONAL: Tor Expert Bundle  (not a Python package, not via pip)
#
# Used as automatic fallback when the primary GMRC endpoint
# (st-gmrc.kur0.deno.dev) is unreachable. Without it the Tor path
# is silently skipped and you fall back to ManifestHub API or
# manual code entry.
#
# Easiest way: run install_online_fix_requirements.bat — it will
# offer to open the Tor download page automatically after pip install.
# Or manually:
#   1. https://www.torproject.org/download/#tor-downloads
#   2. Download "Expert Bundle" (no browser, just the daemon)
#   3. Unzip anywhere, run tor.exe once — it listens on :9050
#   4. Leave it running while using the tool; no other config needed
#
# pysocks (already listed below) handles the SOCKS5 connection.
